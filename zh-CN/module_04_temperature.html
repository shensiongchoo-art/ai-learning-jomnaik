温度（temperature）— 调整创造力

为什么同一个问题有时得到平淡答案，有时又充满创意？来认识模型设置中的“温度”。

核心类比

第 1 幕 — 两位厨师
👨‍🍳
同一个厨房，两位厨师
想象两位厨师接到同一个任务：“请用鸡肉做一道菜。”
稳妥厨师（低温度）：
“好的，我来做马铃薯配烤鸡和肉汁。经典、可靠，正是你预期的味道。”
创意厨师（高温度）：
“来试试香辣芒果鸡肉塔可，淋上青柠椰香酱，再撒焦糖辣椒碎——相信我！”
低温度 = 稳妥且较可预测 高温度 = 更有创意与意外
第 2 幕 — 调节旋钮
🎛️
温度设置
温度是生成时使用的一个数值；不同模型与 API 允许的范围可能不同。
在较低温度下，模型更偏向选择概率较高的下一个词，因此输出通常更稳定。即使设为 0，也不应把所有系统都视为绝对可重复。
在较高温度下，模型会给较少见的候选词更多机会，像增加掷骰子的变化；结果可能更惊喜、更有创意，也更容易偏离主题。
没有绝对好坏，关键是任务需要什么

重点知识

关于温度要记住什么

  • 温度会影响 LLM 输出偏向稳定还是多样
  • 较低温度让高概率词更占优势，输出通常较一致
  • 较高温度让更多候选词有机会被选中，适合创意任务，但不适合借此提高事实准确性
  • 多数聊天产品会自动设置采样参数，用户不一定能直接调节；API 开发者通常拥有更多控制
  • 温度不是事实核查工具。无论设为多少，高风险内容都要验证
