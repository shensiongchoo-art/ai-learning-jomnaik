模块 12

ChatGPT 的工具与记忆

从聊天窗口到工作台：记忆、搜索、数据分析、文件、语音、图像与自定义 GPT

📖 先抓住一个原则
💬
🧠
ChatGPT
Panel 1

ChatGPT 不是只有一个“脑”

ChatGPT 是一个产品界面，背后会接入不同模型与功能。你能用到什么，取决于模型、订阅方案、公司工作区设置和当时是否开启相关工具。

同一句“帮我分析这个表格”，有工具时和没工具时，结果会很不一样。
🧰
🔍
工具箱
Panel 2

工具让它能“做事”

普通 LLM 主要是生成文字。ChatGPT 的工具可以帮它搜索网页、读取文件、做数据分析、理解图片、生成图片，或连接你授权的数据来源。

判断能力时，不只问“模型聪不聪明”，也要问“它现在能用哪些工具”。
🔐
⚙️
控制权
Panel 3

权限和隐私要分清

记忆、文件、联网搜索、连接器都可能涉及个人或公司资料。真正专业的用法，是先确认数据能不能上传、工具有没有开启、输出是否需要人工复核。

敏感资料不要随手丢进去；公司资料尤其要看内部政策。
🧠 记忆怎么理解

上下文 vs 记忆

这两个概念经常被混在一起，但它们不是同一回事。
当前对话上下文（conversation context）
我正在做一个 AI 学习网站，目标读者是中文学习者。
🙋
🤖
在这个对话里，我可以利用你刚给的背景来写内容、改标题、调整难度。
💾 可能保存的记忆：你偏好中英双语解释技术词
↓ 另一个新对话
跨对话记忆（Memory）
帮我写一个 token 的解释。
🙋
🤖
如果记忆功能开启，而且它保存过你的偏好，它可能会主动用中文解释并保留英文术语 token。

🔒 记忆可以在设置中查看、管理或关闭。Temporary Chat 通常用于不想让当前对话影响记忆的场景。

ChatGPT 能力清单

实际可用项会随模型、方案、地区、工作区政策变化。
🔍
搜索
查找近期资讯并给出来源
📊
数据分析
处理 CSV、表格、图表和计算
📄

✨ 多模态体验

文字、图片、语音、文件和代码可以放在同一个工作流里

💬 文字
🖼️ 图片
🎙️ 语音
📊 文件
🐍 代码/计算
🔍 搜索

例子：上传销售表格，请它生成图表；再让它解释异常月份；最后让它草拟给团队的中文总结。这里的关键不是“会聊天”，而是把多个工具串成一个任务流程。

🎯 四个关键能力
🧠

Memory（记忆）

用于跨对话保留偏好或长期背景。它不是当前聊天记录本身，也不代表所有信息都会被永久保存。

可管理
🌐

Search（搜索）

当问题涉及新闻、价格、版本、法规、行程等会变化的信息时，搜索比单靠模型记忆更可靠。

看来源
📊

Data Analysis（数据分析）

可以帮助运行计算、生成图表、检查数据。但你仍要看公式、样本范围、异常值处理和结论是否合理。

要复核
🤖

Custom GPTs（自定义 GPT）

用说明、知识文件和工具配置出特定用途的助手。它通常不是重新训练一个模型，而是给 ChatGPT 一个专门工作方式。

按需配置

🔑 本课重点

📌
先确认能力边界：不同账号、模型、工作区和地区会影响你看到的功能。
🔍
涉及近期事实就要求来源：让它引用搜索结果，并检查来源是否权威。
🔐
敏感资料先问政策：客户资料、公司文件、个人身份信息不要默认上传。
🧪
工具输出仍需验证：图表、计算、代码和总结都可能因为假设错误而出错。
🧠 快速检查

1. ChatGPT 的 Memory 主要解决什么问题？

2. 哪种问题最应该要求 ChatGPT 使用搜索并给来源？

3. 自定义 GPT 通常是什么？
