Claude Projects 与记忆

怎样让 Claude 在持续工作中掌握稳定背景？认识项目知识、项目指令与可控制的项目记忆。

故事开始

第 1 幕 — 自由职业承包商
👷
把 Claude 想成承包商
想象你聘请了一位聪明能干的自由职业承包商，却没有给对方固定的项目档案。每次工作都可能要重新说明背景、规则和关键文件。
普通聊天可以使用账户记忆或搜索过去对话，但这些功能取决于你的设置。若工作有明确边界，Projects 能把相关说明、知识与记忆放在独立空间里。
没有项目空间 = 背景容易分散或混在一起
第 2 幕 — 固定办公桌
🗂️
Claude Projects
Claude Projects 就像给承包商一张固定办公桌。桌上放着项目背景、工作方式、关键文件、术语与风格指南。
新聊天开始时，项目指令和知识仍可供使用；项目记忆启用后，还会维护这个项目独立的摘要。你仍应检查资料是否过时。
Project = 有固定说明、知识与独立记忆边界的工作空间
第 3 幕 — 使用 Projects 前后
💬
同一位用户，两种体验
没有 Projects — 反复补充背景
“你好！我是 Sarah，在槟城经营一家叫 Sweet Crumbs 的小烘焙店。我要写新款芒果蛋糕的 Instagram 文案。我们的语气亲切、有点俏皮，不用正式语言……”
使用 Project — 直接进入任务
“今天要写新款芒果蛋糕的 Instagram 文案。”（项目指令与知识已说明 Sweet Crumbs、受众和品牌语气。）
稳定背景留在项目里，减少重复介绍
第 4 幕 — 办公桌上放什么
📋
项目指令（Project Instructions）
建立 Claude Project 后，可设置项目指令，说明 Claude 在该项目的所有聊天中应如何工作与回答。
项目指令示例
“你正在协助我经营烘焙品牌 Sweet Crumbs。我们位于马来西亚槟城，受众是 20～35 岁年轻人。品牌语气温暖、俏皮，不过分正式。不要使用‘美味’这个词。我们的特色是亚洲与西式融合糕点。”
项目内所有聊天都会使用这组指令

没有 Projects 与使用 Projects

❌ 没有 Projects
↺ 项目资料散落在不同聊天
↺ 业务背景可能与其他工作混在一起
↺ 语气与风格需要另行维护
↺ 关键文件不在统一知识空间
↺ 难以建立清楚的项目边界
✅ 使用 Projects
✓ 项目指令用于每个项目聊天
✓ 业务背景集中保存
✓ 语气与风格要求更一致
✓ 文件留在项目知识库
✓ 更快进入实际任务

Project 里可以放什么

固定办公桌可以放：

📝
项目指令
项目用途、受众、语气、工作规则，以及 Claude 应做或避免的事项
📄
项目知识
上传 PDF、文本、代码与其他支持的资料；Claude 会在项目聊天中使用或检索相关内容
🎨
风格指南
品牌语气、写作样本、术语表与格式规则，帮助输出保持一致
💬
项目聊天与记忆
可搜索项目内过往聊天；启用记忆后，每个项目还有独立记忆摘要，具体取决于账户设置

哪些会保留，哪些会重置

理解项目的记忆边界

保留在 Project 中 ✓

  • 你设置的项目指令
  • 上传到项目知识库的资料
  • 项目中的聊天记录
  • 启用记忆时生成的项目独立摘要

新聊天会重新建立 ↺

  • 当前对话的工作上下文窗口
  • 其他 Project 的内容不会自动混入
  • 项目外聊天与项目记忆相互分开
  • 无痕聊天不会写入聊天历史或记忆

重点知识

关于 Claude Projects 要记住什么

  • Projects 提供独立工作空间，集中保存项目指令、知识、聊天与可选记忆
  • 把稳定背景写进项目指令，不要假设 Claude 会从项目名称或描述自动知道细节
  • 大量项目知识可能通过检索增强生成（RAG）按需取回，而不是每次把所有文件全文放进上下文
  • 项目适合持续性工作，例如业务、写作、研究与重复流程
  • 上下文窗口仍然适用；项目记忆、搜索聊天与无痕模式则有各自设置和边界
