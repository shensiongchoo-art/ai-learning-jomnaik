词元（token）— 语言的乐高积木

LLM 不是按完整单词阅读，而是读取称为“词元”的小块。让我们看看它如何运作。

核心类比

把句子想象成乐高模型。搭建时使用的不是完整单词，而是一块块积木。LLM 会先把句子拆成小块，也就是词元（token），然后才能处理。有些词元是完整单词，有些只是单词的一部分，还有些是标点符号。中文也可能按单字或常见词组拆分，具体方式取决于模型。

实际示例

文字可能如何拆成词元

以下只是近似示意，实际拆分方式因模型而异

文字／短语近似词元数原因

Hello, world!约 4 个词元单词、逗号与感叹号都可能单独计数
人工智能因模型而异可能按单字拆，也可能把常见词组合在一起
1,000 个英文单词 ≈约 750 个词元仅适用于英文的粗略经验值；中文比例不同

为什么这很重要？

🧱 词元与你有什么关系

  • LLM 一次能处理的词元数量有限，这个容量称为上下文窗口（context window），单元 3 会继续讲解
  • 使用按量收费的 LLM API 时，费用通常与输入及输出的词元数有关，所以较长内容可能花费更多
  • 少见词、专业术语和人名往往会被拆成更多词元，可能影响成本与处理效果
  • 理解词元有助于写出更好的提示词：简洁清楚的指令通常更省词元，也更容易得到理想结果
