模块 10 · 第二阶段总结

Claude Code — AI 编程智能体

Claude Code 不只回答问题；在你授予的工具和权限范围内，它能检查项目、编辑代码、运行命令并验证结果。

情境故事

场景 1 — Claude 与 Claude Code
🤖
两种使用方式
Claude 聊天助手擅长写作、分析、解释与讨论；Claude Code 则面向软件开发工作。
可以把前者想成顾问，把 Claude Code 想成会使用终端和项目工具的协作开发者。它能执行多步任务，但实际能力取决于当前环境、可用工具和你批准的权限。
聊天助手偏讨论；Claude Code 偏执行。
场景 2 — 由你设定目标与边界
👨‍💻
Claude Code 工作中
你可以用自然语言说明目标、背景、限制与完成标准。Claude Code 会拟定步骤、修改文件、运行检查，并把结果交给你审阅。
你提出
“在这个示例项目中加入吉隆坡天气页面。先说明方案，只使用免费 API；不要读取密钥。完成后运行测试并给我看差异。”
Claude Code 可以
检查项目 → 提议方案 → 在授权后编辑 HTML 与 JavaScript → 运行测试 → 展示差异 → 根据反馈调整。
你定义结果与边界，并负责最终判断。

关键概念 — 点击展开

🧰
技能（Skills）— 可复用工作手册
把专门说明、参考资料和脚本打包复用
点击展开
🔧 类比：技能像一本放在工具箱里的专业工作手册：需要做某类任务时，智能体读取相应说明，并使用随附的参考资料或脚本。
技能不是“训练进模型里的魔法能力”，而是可复用的任务资源。一个技能可以包含：
  • 完成任务的步骤与质量标准
  • 术语、模板或参考文件
  • 需要时才运行的辅助脚本
  • 关于允许使用哪些工具的说明
实际例子：团队建立“报告审阅”技能，规定先确认来源、抽取表格、标注不确定信息，再输出指定格式。以后执行同类任务时就能沿用这套流程。
🔌
MCP — 工具连接标准
让 AI 应用以统一方式连接外部工具与数据
点击展开
🔌 类比：MCP（Model Context Protocol，模型上下文协议）常被比作 USB-C：它定义一种共同的连接方式，但接上设备不等于自动获得所有权限。
MCP 服务器可以向 AI 应用公开经授权的资源或操作，例如：
  • 搜索内部文档或知识库
  • 查询项目管理系统
  • 在许可范围内读取文件或数据库
  • 调用经过批准的业务操作
实际可做什么由连接器、账户权限、工作区政策和用户批准共同决定。只连接可信服务器，并采用最小权限原则；外部工具返回的内容也可能包含误导性指令。
实际例子：团队把 CRM 的只读搜索工具通过 MCP 提供给智能体。智能体可以查询获准字段，但不能更新记录或发送邮件，除非管理员另外开放这些操作。
🤖
智能体工作流（Agentic workflows）
围绕目标规划、执行、检查并汇报多步任务
点击展开
🏗️ 类比：你向装修承包商说明目标、预算和不能碰的部分；对方拟定工序，并在关键节点请你确认。多步智能体任务也需要清楚边界和检查点。
在智能体工作流中，Claude Code 可以：
  • 接收目标与完成标准
  • 把任务拆成可执行步骤
  • 使用获准工具执行步骤
  • 运行测试或其他验证，并根据结果调整
  • 汇报改动、证据与尚存风险
“智能体式”不代表绝对自主或保证正确。高风险操作应保留人工批准；最终输出仍需审阅。
实际例子：“分析第一季销售数据，生成图表和摘要；不要上传原始客户资料；先给我看分析方法，完成后列出验证步骤。”
💬
清楚指令的组成
不需要特殊语法，但重要任务值得写得具体
点击展开
📋 类比：“弄漂亮一点”很难验收；“保留现有布局，采用高对比配色，手机屏幕可用，并通过无障碍检查”就清楚得多。
一份实用指令通常说明：
  • 目标：要解决什么问题
  • 背景与输入：相关文件、资料或错误信息
  • 限制：不能改变什么，以及安全或技术要求
  • 完成标准：输出形式、测试和可观察结果
  • 检查点：哪些决定必须先获批准
实际例子：“目标：按地区汇总 sales.csv 并生成条形图。限制：不修改原文件，不调用外部服务。完成标准：保存 chart.png，列出计算方法，并检查总数是否与原数据一致。写最终报告前暂停让我审图。”

Claude Code 任务流程

从指令到可审阅结果

1
你说明目标、背景与边界
“分析客户问卷并找出三项主要投诉；移除个人资料，不要上传数据。”
2
Claude Code 检查环境并拟定计划
确认文件与权限，再拆分为：读取 → 清理 → 归类 → 计数 → 汇总
3
它使用获准工具执行
读取指定文件、编写分析代码，并仅在授权时连接外部服务
4
它运行检查并修正明显问题
核对总数、运行测试并说明验证结果；自检能降低风险，但不能取代人工复核
5
你审阅差异、结果与风险
检查报告、图表或代码变更，再决定是否接受、修改或发布

如何给出有效指令

非程序员也能使用的协作清单

✓ 建议这样做
  • 说明目标、相关文件、限制与完成标准
  • 先让它检查现况并解释计划
  • 使用版本控制或备份，并审阅差异
  • 要求运行合适的测试与验证
  • 为敏感或不可逆操作设置批准点
✗ 避免这样做
  • 使用“改好一点”等无法验收的模糊要求
  • 在提示词、文件或日志中贴入密码和 API 密钥
  • 给不受信任工具不必要的写入权限
  • 未审阅便运行或发布生成的代码
  • 把智能体自检当成正确性保证

重点整理

关于 Claude Code，请记住

  • Claude Code 是面向软件开发的编程智能体，能在工具与权限范围内执行多步任务
  • 技能（Skills）把可复用说明、参考资料和脚本组织起来
  • MCP 是连接工具与数据的开放标准；连接能力不等于无限权限
  • 清楚说明目标、背景、限制与完成标准，通常比规定每个技术步骤更实用
  • 采用最小权限、版本控制、检查点、测试与人工审阅
  • 产品功能会更新；请同时查阅 Claude Code 官方文档
