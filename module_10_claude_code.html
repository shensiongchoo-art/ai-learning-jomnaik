Claude Code — Your AI Developer

Claude Code isn't just a chatbot. It's an AI that can plan, build, and execute real tasks — using tools, writing code, and connecting to external systems. Here's how it works.

The Story

Panel 1 — Claude vs. Claude Code
🤖
Two Claudes
You've been talking to Claude the AI assistant — a brilliant conversationalist who can write, analyse, explain, and advise. But there's another version of Claude designed specifically for building things.
Think of it this way: regular Claude is a brilliant consultant who gives you advice and writes reports. Claude Code is a senior developer you can direct to actually build things — and it does the work, step by step, on your computer or in the cloud.
Claude = advisor. Claude Code = builder.
Panel 2 — The Developer You Direct
👨‍💻
Claude Code at Work
You don't give Claude Code code. You give it goals in plain English. It figures out the steps, writes the code, runs it, checks the results, and fixes problems — all on its own.
You tell Claude Code
"Build me a webpage that shows today's weather for Kuala Lumpur, with a nice design."
Claude Code does
Searches for a weather API → writes the HTML and JavaScript → connects to the API → tests it → shows you the result → adjusts the design based on your feedback.
You describe the outcome. It handles the execution.

The Key Concepts — Click Each to Explore

🧰
Skills — The Toolbox
Reusable abilities Claude Code can pick up and use
Click to expand
🔧 Analogy: Imagine a developer's toolbox. Each tool is a skill — a web scraper, a PDF reader, a chart builder, a data analyser. Claude Code can pick up whichever tool a task needs, use it, and put it down.
Skills are pre-built capabilities that Claude Code can deploy. Instead of writing everything from scratch each time, it can call on skills it already knows:
  • Read and extract data from PDFs
  • Scrape information from websites
  • Generate charts and visualisations
  • Search academic papers or databases
  • Write and run code in Python, R, or shell
In practice: You say "Summarise this PDF report." Claude Code uses its PDF reading skill to extract the text, then its summarisation ability to produce a clean summary — without you specifying how.
🔌
MCP — The USB Ports
How Claude Code connects to external tools and services
Click to expand
🔌 Analogy: Your laptop has USB ports. You can plug in a keyboard, a mouse, an external hard drive, a microphone — whatever you need. MCP (Model Context Protocol) is Claude Code's USB standard. It lets developers "plug in" new tools and services.
MCP is a protocol — a standardised way for Claude Code to connect to external tools:
  • Connect to your company's database
  • Read and write files on your computer
  • Send emails or calendar invites
  • Search Google or internal documentation
  • Control software on your machine
A developer "plugs in" a new MCP tool, and Claude Code can now use that service — just like plugging in a new USB device.
In practice: A developer adds an MCP connector for your company's CRM. Now Claude Code can read customer records, update them, and send follow-up emails — all directed by plain English instructions.
🤖
Agentic Workflows — The Autonomous Builder
When Claude Code plans and executes multi-step tasks on its own
Click to expand
🏗️ Analogy: You hire a contractor to renovate your kitchen. You tell them what you want — modern, open-plan, white cabinets — and they figure out the sequence: demo, electrical, plumbing, tiling, cabinets, paint. They report back at each stage. You're not writing the step-by-step instructions. That's an agentic workflow.
In an agentic workflow, Claude Code:
  • Receives a high-level goal ("Build me a sales report")
  • Breaks it into steps autonomously
  • Executes each step — fetching data, writing code, running it
  • Checks results and adjusts
  • Reports back to you with the finished output
You stay in control at key decision points — but you're not writing every step manually.
In practice: "Analyse our sales data for Q1 and make a summary with charts." Claude Code reads the data files, writes analysis code, generates charts, spots the key trends, and delivers a formatted report — all from that one instruction.
💬
Syntax — How Developers Talk to Claude Code
The patterns and structure behind effective Claude Code instructions
Click to expand
📋 Analogy: A contractor needs a good brief. "Make it nice" → poor brief. "White cabinets, quartz countertops, pendant lights above the island, under 50k budget, done by March" → great brief. Claude Code responds the same way — better structure = better results.
Developers use consistent patterns when giving Claude Code instructions:
  • Steps: Define discrete tasks (fetch data → analyse → chart → report)
  • Inputs: Specify what data or files to work with
  • Outputs: Define what deliverable to produce (CSV, chart, PDF)
  • Constraints: Set limits ("use only free APIs", "keep files under 5MB")
  • Checkpoints: Ask Claude Code to pause and show results before continuing
In practice: Instead of "analyse my data", a developer writes: "Step 1: Load sales.csv. Step 2: Group by region and calculate totals. Step 3: Create a bar chart. Step 4: Save as chart.png. Pause and show me the chart before writing the final report." This gives Claude Code clear structure to work within.

How a Claude Code Task Flows

From your instruction to finished output

1
You give a goal in plain English
"Analyse our customer survey responses and identify the top 3 complaints."
2
Claude Code makes a plan
Breaks the goal into steps: read file → process text → identify themes → count frequency → summarise
3
It uses its skills and MCP tools
Reads the survey file, runs analysis code, connects to any needed external services
4
It checks and fixes its own work
Reviews the output for quality, corrects errors, and tries again if something doesn't look right
5
It delivers the result to you
A finished report, chart, or file — and an explanation of what it found and how

How to Give Claude Code Good Instructions

Your non-coder guide to directing Claude Code

✓ Do this
  • Describe the outcome you want, not the technical steps
  • Specify the file or data it should work with
  • Say what format you want the output in
  • Ask it to pause and show you results before proceeding
  • Break big tasks into smaller goals if you're unsure
✗ Avoid this
  • Vague instructions ("make it better", "do something with the data")
  • Assuming it knows which file to use — tell it explicitly
  • Skipping the review step on important outputs
  • Asking it to do 10 things in one sentence
  • Forgetting it can make mistakes — always check the result

Key Facts

What to remember about Claude Code

  • Claude Code is an agentic AI — it doesn't just answer questions, it plans and executes multi-step tasks
  • Skills = pre-built capabilities Claude Code can use (like tools in a toolbox)
  • MCP = the protocol for connecting Claude Code to external services (like USB ports)
  • Agentic workflows = Claude Code autonomously handles a sequence of steps toward a goal
  • You give it a goal in plain English — it figures out how to achieve it
  • As a non-coder, you can direct Claude Code effectively by being clear about goal, inputs, and expected output
