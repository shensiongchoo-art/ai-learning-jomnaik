The Context Window

Why does the LLM sometimes forget what you said earlier? Meet the memory tank.

The Story

Panel 1 — The Goldfish Problem
🐟
The LLM's Memory
You might have heard the myth that goldfish have a 3-second memory. That's not quite true for fish — but it's a great way to think about LLMs.
An LLM doesn't have a permanent memory. Every conversation lives inside a tank. When the tank is full, old messages fall out — the LLM simply can't see them anymore.
Key concept: the tank = context window
Panel 2 — What's In the Tank?
💬
Your Conversation
Every message in your conversation takes up space in the tank — your messages AND the LLM's replies. Even the instructions you give at the start live in the tank.
You said (message 1)
"My name is Sarah. I'm a teacher."
LLM replied
"Hi Sarah! Great to meet a teacher. How can I help?"
Both messages now occupy tank space
Panel 3 — The Tank Gets Full
🌊
Tank Overflow!
After many messages, the tank is full. The oldest messages — including "My name is Sarah" — fall out.
You ask (message 47)
"Can you use my name in your reply?"
LLM replies (confused)
"I don't see your name in our conversation. Could you remind me?"
It's not being rude — it genuinely can't see that far back!
Panel 4 — Different Tanks for Different Models
🪣
Tank Sizes Vary
Different LLMs have different tank sizes. Some models can hold the equivalent of a short email. Others can hold an entire novel.
The size is measured in tokens (remember those Lego bricks from Module 2!). A common size is 128,000 tokens — roughly a full novel.
Bigger tank = longer conversations without forgetting

Key Facts

What to remember about the Context Window

  • The context window is the LLM's short-term memory — everything it can "see" in a conversation
  • It includes your messages AND the LLM's replies — both eat up tank space
  • When it's full, the oldest messages are dropped silently — the LLM won't warn you
  • Starting a new chat = emptying the tank. All memory is gone
  • Some tools (like Claude Projects) save context across sessions — but that's an add-on, not the default
